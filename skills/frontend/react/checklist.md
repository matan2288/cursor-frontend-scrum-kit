# Component Checklist

- Props are typed and named consistently.
- Required and optional behavior is clear from defaults.
- Event handlers match local naming and controlled/uncontrolled patterns.
- Loading, empty, disabled, selected, open, error, and success states are explicit when relevant.
- Markup is semantic and accessible (labels, focus, keyboard).
- Styling matches nearby components and stays responsive.
- Async work avoids avoidable waterfalls.
- Imports do not pull unnecessary bundle weight.
- No unrelated refactors or dependency changes are included.
